Variables let y Constantes const

Variables LET

Al usar TypeScript, lo que debemos hacer es reemplazar var por let al instanciar una variable.

Las variables let en un archivo .ts sirve para evitar sobreescribir valores de variables en un scope diferente.

Si TypeScript identifica variables con el mismo nombre en diferente scope(bloque) al momento de compilar renombra la variable en el archivo .js

Constantes CONT

Al igual que let, const es una palabra reservada solo para TypeScript.

Por convención las constantes siempre se escriben en mayuscula. ej. const GENERO = "Femenino"

Las constantes al momento de instanciarla se le debe asignar el valor.

Una vez se inicializa o ejecuta el script, su valor no se puede cambiar.

Al momento de compilar, el archivo .js renombra el nombre de la variable en caso de encontrar variables constantes en diferentes scope con el mimsmo nombre (lo mismo que let)

Tipos de datos

TypeScript identifica que tipo de dato tiene una variable, esto lo realiza al momento de asignarle el primer valor a dicha variable.

TypeScript nos muestra un mensaje de advertencia cuando se cambia el tipo de dato de una variable, si lo quisieramos dejar asi no hay problema al momento de compilar y generar el .js

La forma correcta de instanciar una variable es indicando que tipo de dato permitira: ej. let nombre:string = "Hola Mundo"

Los tipos de datos mas comunes son:
let xxxx:string
let xxxx:number
let xxxx:boolean
let xxxx:date

TypeScript tambien tiene otros tipos de datos especiales, como lo es el tipo de dato Any, esto significa que una variable de tipo any permite cualquier valor (Con esto evitamos el mensaje de advertencia que genera TypeScript)

Ver mas: https://code.tutsplus.com/es/tutorials/typescript-for-beginners-basic-data-types--cms-29438

Template literales

Se puede decir que los templates literales son cadenas de texto donde imprimimos valores de variables o realizamos saltos de linea.

Para usar alguna variable dentro del template literal se debe hacer de la siguiente manera: $( nombre_variable )

Para hacer uso de esto, debemos reemplazar en una cadena de texto los tipicos carateres de "" o '' y usar ``

No solo se pueden usar variables, tambien se puede imprimir resultados de una funcion (return) o realizar operaciones. $(fun_saludar() ) || $( 1+1 )

Ver mas: https://developer.mozilla.org/es/docs/Web/JavaScript/Referencia/template_strings

Parametros de una función

Las funciones pueden recibir 3 tipos de parametros diferentes

Los parametros pueden ser obligatorios, opcionales o por defecto

function activar(quien:string,objeto:string = "batiseñal",momento?:string)
{

quien es un parametro obligatorio

objeto es un parametro por defecto (su valor por defecto es batiseñal)

momento? es un parametro opcional, el simbolo ? lo indica

}

Funciones flecha

La estructura de una funcion flecha es let nombreFuncion = () => "hola Mundo"

Los parametros van dentro de () ejemplo: let nombreFuncion = (a,b) => a+b

Si la funcion tiene un solo parametro se puede omitir los parentesis, si no tiene parametros se ponen los parentesis vacios

El operador => equivale al return

Si despues del operador => tiene mas de una linea se debe usar llaves { muchas lineas }

Las funciones flechas son muy utiles cuando tenemos fuciones dentro de otras funciones, especialmente por el uso del this dentro de los diferentes scope

Destructuración de Objetos

Destructurar es obtener en variables los valores de sus propiedades

La estructura es: let {propiedades} = objeto

Las propiedades estan dentro de llaves y separadas por una coma {prop4, prop6, ...., prop1}

Las propiedades se les puede cambiar el nombre (alias) al momento de destructurarlas let {prop1:nombre, prop5:apellidos} = objeto

Destructuración de Arreglos

Destructurar es obtener en variables los valores del arreglo, se obtienen en forma secuencial

La estructura es: let [posicion0, posicion1, ..... posicionN] = arreglo

Si solo queremos el valor de la posicion 2 se debe dejar espacio vacio separado por comas hasta llegar a la posicion deseada: [ , ,posicion2] = arreglo

Promesas (Promise)

Las promesas permiten ejecutar una tarea (codigo o funcion) cuando una tarea asincrona es terminada

Las promesas tiene una estructura que requiere dos parametros que indican que hacer cuando se cumple la condicion y cuando genera error

Estos 2 parametros por convención se conocen como (resolve y reject)

La estructura base es:

let prom1 = new Promise(function(resolve, reject){
________ Aqui va el codigo donde requerido el cual indica si es resolve o reject
________ resolve() Se hace el llamado a resolve() cuando es exitoso
________ reject() Se hace el llamado a reject() cuando salio algo mal
}

Para llamar una promesa se utiliza el metodo then de la promesa el cual espera de igual forma 2 parametros (correco , incorrecto)

Interfaz (Interface)

Una interface es como una plantilla similar a una clase con propiedades que se utiliza para indicar que se debe cumplir su formato, se usa como si fuera un tipo de dato

Para crear una interfaz se utiliza la palabra reservada interface y el nombre que le queremos dar, luego dentro de llaves van las propiedades

interface Xmen {
nombre:string,
poder:string
}

Su funcionalidad se ve al validar el contenido de la variable asignandole como tipo de dato el nombre de la interfaz que creamos

function enviarMision(xmen:Xmen){ console.log("Enviar a mision "+xmen.nombre) }

Introducción POO

POO: Es una serie de normas establecidas que permiten tener una estructura clara para programar y legible para cualquier programador.

Clase (Objeto): Es algo que existe en la vida real. Una clase puede tener muchas propiedades. ej. Un Carro

Propiedades (Caracteristicas): Son las cosas que tiene una Clase Puertas, Color, Llantas

Metodos (Acciones/Funciones): Son las cosas que puede hacer una Clase eje. Encender, Apagar, Acelerar, Frenar

Concepto de Herencia: Es cuando una clase hereda las propiedades de otra clase


Constructor

Un constructor es una funcion(metodo) de una clase que se ejecuta siempre y unicamente al momento de instanciar la clase

En typescript se crea la funcion con la palabra reservada constructor

constructor(){
Aqui va el codigo del constructor
}

El constructor puede recibir parametros, los parametros se enviando al momento de instanciar la clase ej. let carro = New Carro(color,placa,cant_puertas){}
